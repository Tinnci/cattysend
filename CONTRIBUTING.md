# Contributing to Cattysend

We love your input! We want to make contributing to `cattysend` as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Workflow

1. **Fork the repo** and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`cargo test`).
5. Make sure your code lints (`cargo clippy`).
6. Format your code (`cargo fmt`).

## Pull Request Process

1. Provide a clear description of the problem and solution.
2. Link to any relevant issues.
3. Verify that your code builds and passes all checks.
4. We will review your PR as soon as possible.

## Any Questions?

Feel free to open an issue to discuss your ideas or questions.
